DataFrame Transformations
The Operations You'll Use Every Day

select, filter, withColumn, drop, rename — and chaining like a pro

// Transformations vs Actions — A Reminder

Transformations like select(), filter(), and withColumn() are lazy — they build the execution plan. Nothing runs until you call an action like show(), count(), or write(). Chain as many transformations as you need — Spark optimizes them together.

// Core Transformation Cheatsheet
select()

Pick specific columns. Supports column names, expressions, and aliases.

filter() / where()

Filter rows by condition. Both are identical — use whichever reads cleaner.

withColumn()

Add a new column or overwrite an existing one with an expression.

withColumnRenamed()

Rename a column without touching others.

drop()

Remove one or more columns from the DataFrame.

distinct()

Remove duplicate rows. Equivalent to SQL's SELECT DISTINCT.

limit()

Return only the first N rows. Useful for sampling large datasets.

orderBy() / sort()

Sort rows by one or more columns, ascending or descending.

// All Transformations in Action
day04_transformations.py 
from pyspark.sql import SparkSession
from pyspark.sql import functions as F

spark = SparkSession.builder.appName("Day04").getOrCreate()

data = [
    (1, "Alice",   "Engineering", 95000.0),
    (2, "Bob",     "Marketing",   72000.0),
    (3, "Charlie", "Engineering", 88000.0),
    (4, "Diana",   "HR",          81000.0),
    (5, "Eve",     "Marketing",   67000.0),
]
df = spark.createDataFrame(data, ["id", "name", "department", "salary"])

# ── select: pick columns + alias ──────────────────────
df.select(
    F.col("name"),
    F.col("salary").alias("annual_salary"),
    F.col("department").alias("dept")
).show()

# ── filter: keep rows matching condition ───────────────
df.filter(F.col("department") == "Engineering").show()
df.filter(F.col("salary") > 80000).show()

# ── Multiple conditions ────────────────────────────────
df.filter(
    (F.col("department") == "Engineering") &
    (F.col("salary") > 85000)
).show()

# ── withColumn: add/overwrite a column ────────────────
df = df.withColumn("salary_usd", F.col("salary") * 1.0)
df = df.withColumn("bonus",      F.col("salary") * 0.10)
df = df.withColumn("seniority",
    F.when(F.col("salary") >= 90000, "Senior")
     .when(F.col("salary") >= 75000, "Mid")
     .otherwise("Junior")
)

# ── rename, drop, sort ────────────────────────────────
df = df.withColumnRenamed("name", "full_name")
df = df.drop("salary_usd")
df = df.orderBy(F.col("salary").desc())

# ── Chain everything together (the clean way) ─────────
result = (
    df
    .filter(F.col("salary") > 70000)
    .select("full_name", "department", "salary", "seniority")
    .orderBy(F.col("salary").desc())
    .limit(3)
)
result.show()

spark.stop()
✦ Always use F.col() over string column names. Writing F.col("salary") > 80000 is safer than "salary > 80000" — it gives you compile-time errors, IDE autocomplete, and works correctly across all contexts.
// Key Takeaways
  • All transformations are lazy — chain them freely, Spark optimizes the full plan before running.
  • filter() and where() are identical — pick whichever reads more naturally.
  • Use F.when().when().otherwise() to build conditional columns — it's your Spark equivalent of SQL CASE WHEN.
  • Combine & (and) / | (or) for multi-condition filters — always wrap each condition in parentheses.
  • Chain transformations in a single block with parentheses — it keeps your code readable and avoids unnecessary intermediate DataFrames.
