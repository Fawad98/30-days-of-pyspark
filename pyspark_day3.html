Reading & Writing Data
Every Format You'll Use in Production

Parquet, CSV, JSON, Delta — and partitioning strategies that scale

// Choosing the Right Format
Parquet
✦ Production Standard

Columnar, compressed, schema-embedded. Best for analytics workloads. Read only the columns you need.

Delta Lake
✦ Modern Lakehouse

Parquet + ACID transactions + time travel. The go-to format for reliable data lakes.

JSON
Common

Great for semi-structured / nested data. Verbose and slow compared to columnar formats.

CSV
Legacy / Ingestion

Universal but inefficient. Use only at ingestion boundaries — convert to Parquet ASAP.

// Reading Data
day03_read_write.py 
from pyspark.sql import SparkSession
from pyspark.sql.types import StructType, StructField, StringType, DoubleType, IntegerType

spark = SparkSession.builder.appName("Day03").getOrCreate()

schema = StructType([
    StructField("id",         IntegerType(), False),
    StructField("name",       StringType(),  False),
    StructField("department", StringType(),  True),
    StructField("salary",     DoubleType(),  True),
])

# ── Read CSV ───────────────────────────────────────────
df_csv = spark.read \
    .schema(schema) \
    .option("header", True) \
    .option("nullValue", "NA") \
    .csv("data/employees.csv")

# ── Read JSON ──────────────────────────────────────────
df_json = spark.read \
    .schema(schema) \
    .json("data/employees.json")

# ── Read Parquet (fastest for analytics) ──────────────
df_parquet = spark.read \
    .parquet("data/employees.parquet")

# ── Read Delta Lake ────────────────────────────────────
df_delta = spark.read \
    .format("delta") \
    .load("data/employees_delta")
// Writing Data with Partitioning
day03_write.py 
# ── Write modes: overwrite | append | ignore | error ──

# Write as Parquet — partitioned by department
df_csv.write \
    .mode("overwrite") \
    .partitionBy("department") \
    .parquet("output/employees_parquet")

# Output folder structure:
# output/employees_parquet/
# ├── department=Engineering/
# ├── department=Marketing/
# └── department=HR/

# Write as Delta (append new records)
df_csv.write \
    .format("delta") \
    .mode("append") \
    .partitionBy("department") \
    .save("output/employees_delta")

# Write single CSV file (coalesce avoids 100s of part files)
df_csv.coalesce(1) \
    .write \
    .mode("overwrite") \
    .option("header", True) \
    .csv("output/employees_single")

spark.stop()
✦ Partitioning Strategy: Partition by columns you frequently filter on (e.g. department, year, country). Spark will skip entire partitions it doesn't need — this is called partition pruning and can cut query time dramatically.
⚠️ Watch out: Never partition by a high-cardinality column like user_id or timestamp — you'll end up with millions of tiny files and kill your cluster.
// Key Takeaways
  • Use Parquet as your default format — it's columnar, compressed, and fast.
  • Use Delta Lake when you need ACID transactions, upserts, or time travel.
  • Convert CSVs to Parquet at ingestion — never keep raw CSVs in your pipeline.
  • partitionBy() enables partition pruning — always partition by low-cardinality filter columns.
  • coalesce(1) before writing a CSV avoids the dreaded part-00000 file explosion.
  • Write modes: overwrite, append, ignore, error — know when to use each.
