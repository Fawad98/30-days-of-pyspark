Introduction to PySpark
& the Spark Architecture

Your foundation for distributed data engineering at scale

// What is PySpark?

PySpark is the Python API for Apache Spark — an open-source, distributed computing engine designed to process massive datasets across a cluster of machines. It brings the expressiveness of Python to Spark's in-memory computation engine.

// Core Architecture Concepts
🧠

Driver Program

The master node. Runs your main() function, creates the SparkContext, and coordinates all tasks.

⚙️

Executors

Worker JVMs on cluster nodes. They run tasks assigned by the driver and store data partitions.

📦

RDD / DataFrame

Core data abstractions. DataFrames are the modern, optimized way to work with structured data.

🔁

Lazy Evaluation

Transformations don't execute immediately — Spark builds a DAG and only runs on an action.

// Your First PySpark Session
day01_intro.py 
# Install: pip install pyspark

from pyspark.sql import SparkSession

# SparkSession is the unified entry point in Spark 2.0+
spark = SparkSession.builder \
    .appName("Day01 - Intro to PySpark") \
    .master("local[*]") \
    .getOrCreate()

# Create a simple DataFrame from Python list
data = [
    ("Alice",   28, "Engineering"),
    ("Bob",     35, "Marketing"),
    ("Charlie", 22, "Engineering"),
    ("Diana",   30, "HR"),
]
columns = ["name", "age", "department"]

df = spark.createDataFrame(data, columns)

# Actions — these trigger Spark to actually execute
df.show()          # Print rows as table
df.printSchema()   # Inspect column types
print(df.count())  # Count rows → 4

# Always release cluster resources
spark.stop()
// Expected Output
+-------+---+-----------+ | name|age| department| +-------+---+-----------+ | Alice| 28|Engineering| | Bob| 35| Marketing| |Charlie| 22|Engineering| | Diana| 30| HR| +-------+---+-----------+ root |-- name: string (nullable = true) |-- age: long (nullable = true) |-- department: string (nullable = true) 4
// Key Takeaways
  • PySpark lets you write Python while Spark handles distributed execution under the hood.
  • SparkSession is your single entry point — always build it first.
  • local[*] runs Spark on all your CPU cores — great for local development.
  • Transformations are lazy — .show() and .count() are actions that trigger the DAG.
  • Always call spark.stop() to free resources when you're done.
